01

Visão do Produto

O Aproove é uma plataforma SaaS B2B que centraliza o fluxo de planejamento, produção, aprovação e publicação automatizada de conteúdo para redes sociais, eliminando o caos operacional de aprovações via WhatsApp, e-mail e PDFs.

Missão

Transformar o processo de aprovação de conteúdo de marketing de uma cadeia desorganizada em um fluxo rastreável, previsível e automatizado.

Visão de Longo Prazo

Tornar-se a plataforma central de operação para agências de marketing e departamentos de marketing corporativo no Brasil e América Latina.

Proposta de Valor Central

Reduzir retrabalho, acelerar aprovação, dar previsibilidade operacional ao marketing e eliminar publicações erradas por falta de aprovação registrada.

Problema e Solução

O Problema Real

Como o processo funciona hoje

1
Briefing por mensagem
WhatsApp, e-mail, ligação — sem registro estruturado
2
Arte enviada por Drive ou WeTransfer
Sem versionamento, sem contexto vinculado
3
Aprovação informal
"Ok" no WhatsApp, print de tela, áudio de 2min
4
Publicação manual
Social media publica na memória — sem garantia de data/hora
!
Retrabalho e erro
Versão errada publicada, cliente insatisfeito, agência perde tempo

Como o Aproove resolve

1
Campanha estruturada
Planejador define redes, formatos, datas e responsáveis
2
Produção organizada
Social media e designer têm tarefas claras e prazos
3
Mockup automático
Cliente vê o post exatamente como será publicado
4
Aprovação registrada
Clique no botão "Aprovar" — com data, hora e usuário
✓
Agendamento automático
Aprovado → publicado na data certa, sem intervenção manual

Empresas médias com departamento de marketing próprio, que precisa aprovar conteúdo com diretoria, jurídico ou cliente interno. Necessita rastreabilidade e histórico para auditoria.

Fluxos Completos do Produto

Fluxo 1 — Planejamento da Campanha

Ator: Planejador / Gestor de Marketing  |  Objetivo: Estruturar a campanha antes da produção iniciar.

1
Criar nova campanha
Nome, cliente associado, período (início/fim), objetivo geral
2
Selecionar redes sociais
Instagram, LinkedIn, TikTok, Facebook, X — múltipla seleção
3
Definir formatos por rede
Post estático, carrossel, story, reels, vídeo — por rede selecionada
4
Definir objetivo do conteúdo
Awareness, Engajamento, Geração de lead, Institucional, Comercial
5
Atribuir responsáveis e datas
Social media responsável + designer responsável + data de publicação por peça
6
Salvar planejamento
Sistema gera lista de ativos a produzir — campanha criada e rastreável

Fluxo 2 — Produção de Conteúdo

Atores: Social Media + Designer  |  Objetivo: Transformar planejamento em ativos prontos para aprovação.

1
Social Media recebe tarefa
Notificação da peça atribuída — rede, formato, data e objetivo
2
Preenchimento do copy
Legenda, CTA, hashtags, texto alternativo — campo por campo
3
Designer recebe briefing
Vê o copy aprovado + instruções de arte vinculadas à peça
4
Upload da arte
Imagem ou vídeo conforme formato + dimensões exigidas pela rede
5
Revisão interna
Gestor de marketing valida internamente antes de enviar ao cliente
6
Peça marcada como pronta para cliente
Status atualizado — aguardando aprovação externa

Fluxo 3 — Geração de Mockup

Ator: Sistema  |  Objetivo: Apresentar ao cliente a peça no contexto visual da rede social.

1
Arte + copy vinculados
Sistema detecta que ambos os elementos estão completos
2
Geração automática do preview
Sistema renderiza o post no frame da rede social correspondente
3
Mockup apresentado ao cliente
Layout da rede + legenda + CTA + sequência (carrossel) + data programada

Fluxo 4 — Aprovação do Cliente

Ator: Cliente externo  |  Objetivo: Aprovar ou solicitar ajustes com registro formal.

1
Cliente recebe link
E-mail com link de aprovação — acesso público ou autenticado
2
Visualização do mockup
Tela simples e limpa — foco total na peça em contexto
3
Decisão do cliente
Três opções: Aprovar / Solicitar ajuste (com comentário) / Reprovar
4
Registro automático
Data, hora, usuário e comentário gravados — imutável

Fluxo 5 — Ciclo de Revisão

Atores: Time interno + cliente  |  Objetivo: Garantir controle de versão no retrabalho.

1
Peça retorna ao time
Comentário do cliente vinculado à peça — visível para social media e designer
2
Ajuste de copy e/ou arte
Nova versão criada — versão anterior preservada no histórico
3
Nova versão para o cliente
Cliente pode comparar versão atual vs. versão anterior
4
Ciclo se repete até aprovação
Cada rodada de revisão é registrada — histórico completo disponível

Fluxo 6 — Agendamento Automático

Ator: Sistema  |  Objetivo: Eliminar a etapa manual de publicação após aprovação.

1
Cliente aprova
Aprovação registrada com timestamp
2
Sistema verifica data e rede
Consulta a data programada no planejamento da campanha
3
Peça entra na fila de agendamento
Integração com API da rede social — publicação programada
4
Status atualizado
Aprovado → Agendado → Publicado — visível no dashboard

O MVP do Aproove é a versão mais enxuta que já gera valor comercial real: o cliente aprova por uma tela simples, a equipe interna rastreia status, e o post é agendado automaticamente. Tudo o que vai além disso é fase 2 ou visão futura.

✅ Essencial — MVP
  • Cadastro de campanha
  • Seleção de redes e formatos
  • Upload de arte e copy
  • Mockup visual por rede
  • Portal de aprovação do cliente
  • Comentário em reprovação
  • Histórico de versões

Wireframes Conceituais

Tela 1 — Dashboard Principal

Objetivo da tela

Dar ao gestor visão gerencial imediata de todas as campanhas e gargalos ativos.


Blocos: Barra lateral com navegação (Campanhas / Peças / Clientes / Configurações) · Header com nome da conta e usuário logado · KPIs: Total de peças ativas / Pendentes de aprovação / Em ajuste / Agendadas · Lista de campanhas ativas com barra de progresso por status · Atalho rápido para criar nova campanha.


Ações disponíveis: Criar campanha · Filtrar por status · Filtrar por rede · Ver detalhes de campanha.

Tela 2 — Nova Campanha

Objetivo da tela

Criar uma nova campanha estruturada com todos os parâmetros necessários.


Campos: Nome da campanha · Cliente · Data de início · Data de fim · Objetivo da campanha (dropdown) · Seleção de redes (checkboxes visuais com ícone de cada rede) · Para cada rede: seleção de formato(s) + data de publicação + responsável social media + responsável designer.


Resultado: Campanha criada com lista de peças gerada automaticamente, cada uma com status "Em Briefing".

Tela 3 — Detalhe da Peça (Produção)

Objetivo da tela

Permitir que social media e designer completem a produção da peça.


Campos: Rede social + formato (exibido) · Data de publicação · Status atual (badge) · Área de copy: legenda, CTA, hashtags · Upload de arte (drag & drop) · Pré-visualização da arte enviada · Botão de marcar como "Pronto para revisão interna".

Tela 4 — Portal do Cliente (Aprovação)

Objetivo da tela

Tela pública ou com login simples — experiência focada em aprovação, sem ruído.


Blocos: Logo da agência (personalizado) · Nome da campanha + data de publicação · Mockup visual simulando a rede social (frame da rede + arte + copy) · Botões de ação: [Aprovar] [Solicitar Ajuste] · Campo de comentário (aparece se clicar em ajuste) · Histórico de versões anteriores (opcional no MVP).


Regras: Após aprovação, botões desaparecem · Comentário é obrigatório para solicitar ajuste · Aprovação grava timestamp imutável.

Tela 5 — Histórico de Versões

Objetivo da tela

Permitir auditoria completa do ciclo de aprovação de uma peça.


Blocos: Timeline vertical da peça · Por versão: número da versão + data + quem enviou + arte (thumbnail) + copy · Comentários do cliente por versão · Status de cada versão · Versão atual destacada.

LGPD e Segurança

Dados Pessoais Coletados
  • Nome e e-mail de usuários
  • Dados de acesso (login)
  • IP e logs de sessão
  • Arquivos de peças (imagens/vídeos)
  • Comentários e feedbacks
  • Registro de aprovações
Obrigações LGPD
  • Política de Privacidade clara
  • Termo de Uso por usuário
  • Consentimento explícito
  • Direito à exclusão de dados
  • Portabilidade de dados
  • DPO (Encarregado de Dados)
Medidas de Segurança
  • Autenticação via JWT + OAuth
  • HTTPS obrigatório em todas as telas
  • Criptografia em repouso (AES-256)
  • Logs de auditoria por ação
  • Controle de acesso por papel (RBAC)
  • Backup automático diário

Regra de Ouro — Aprovação como Registro Legal

O registro de aprovação do cliente (timestamp + usuário + IP) deve ser tratado como um documento com valor probatório. Nunca pode ser editado, deletado ou sobrescrito após confirmação. Este é um diferencial competitivo e uma proteção legal para agências e seus clientes.

Narrativa para Investidores

Tese de Investimento em Uma Frase

O Aproove resolve um problema crônico e universal de agências de marketing: a aprovação de conteúdo acontece hoje por WhatsApp — e isso custa retrabalho, atrasos, erros de publicação e clientes insatisfeitos.

Tamanho do Mercado

Brasil: ~15.000 agências de marketing digital ativas · ~100.000 empresas com equipe interna de marketing. Ticket médio estimado R$ 500/mês por cliente.


TAM: R$ 1,1 bi/ano · SAM (agências): R$ 90 mi/ano · SOM (5% SAM): R$ 4,5 mi/ano em 3 anos.

Diferencial Competitivo
  • Único foco em fluxo de aprovação com mockup
  • Registro legal de aprovação (timestamp + IP)
  • Experiência radicalmente simples para o cliente
  • Agendamento automático pós-aprovação
  • Sem curva de aprendizado para cliente final

IMPORTANT: All user interface text, labels, buttons, messages, and content must be written in Brazilian Portuguese (pt-BR). --- PROJECT OVERVIEW Aproove centralizes the content approval flow between marketing teams and their clients. The core problem it solves: today, content approvals happen via WhatsApp, email, and PDFs — causing rework, delays, misalignment, and publication errors. --- TECH STACK - Frontend: React + TypeScript + Tailwind CSS - Backend: Node.js + Express (or Next.js API routes) - Database: Supabase (PostgreSQL) or Firebase - Storage: Supabase Storage or Cloudinary (for image/video uploads) - Auth: Supabase Auth or Clerk - Deployment: Replit (web) --- USER ROLES 1. Internal Team (authenticated): - Planner: creates campaigns, defines networks/formats/dates - Social Media: writes copy (caption, CTA, hashtags) per piece - Designer: uploads artwork per piece - Marketing Manager: internal review and approval - Admin: manages users, clients, integrations 2. External User (client): - Client Approver: accesses via public link or simple login, views mockup, approves or requests changes --- CORE FEATURES TO BUILD (MVP SCOPE) 1. CAMPAIGN PLANNING Screen: "Nova Campanha" (New Campaign) Fields: - Campaign name - Associated client - Start and end date - Campaign objective (dropdown: Awareness / Engagement / Lead Generation / Institutional / Commercial) - Social networks (multi-select checkboxes with icons): Instagram, LinkedIn, TikTok, Facebook, X/Twitter - For each selected network: format(s) + publication date + assigned social media user + assigned designer On save: system auto-generates a list of content pieces with status "Em Briefing" (In Briefing) 2. CONTENT PRODUCTION Screen: "Detalhe da Peça" (Content Piece Detail) Fields: - Social network + format (displayed, not editable) - Publication date - Status badge - Copy area: caption, CTA, hashtags (filled by Social Media role) - Artwork upload (drag & drop, image or video) — filled by Designer role - Artwork preview after upload - Button: "Marcar como pronto para revisão" (Mark as ready for review) 3. VISUAL MOCKUP GENERATION When both copy and artwork are complete, the system renders a visual mockup simulating the post as it would appear on the social network: - Instagram: white phone frame, profile picture placeholder, username placeholder, image, caption, like/comment bar - LinkedIn: feed-style card with profile photo, name, text, image - TikTok: vertical dark phone frame with overlay text, like/share buttons - Facebook: standard feed post card - X/Twitter: tweet card with text and image 4. CLIENT APPROVAL PORTAL Screen: public or lightly-authenticated page accessible via unique link Layout: - Agency logo (customizable) at top - Campaign name + scheduled publication date - Visual mockup (network-specific frame) - Caption text displayed below - Three action buttons: [✓ Aprovar] [✎ Solicitar Ajuste] [✗ Reprovar] - Comment field (mandatory, appears when "Solicitar Ajuste" is clicked) - After approval: buttons disappear, confirmation message shown On approval: timestamp, user identifier, and IP are recorded (immutable) On revision request: comment is attached to the piece; status changes to "Em Ajuste" 5. VERSION HISTORY Screen: "Histórico de Versões" (Version History) - Vertical timeline per content piece - Each version shows: version number, date, who submitted, artwork thumbnail, copy text, client's comment (if any), status badge - Current version highlighted 6. AUTOMATIC SCHEDULING Logic (backend): - On client approval, system reads the scheduled publication date from campaign planning - Status updates automatically: Aprovado → Agendado - (For MVP: scheduling confirmation is visual only — no real API integration with social networks yet) - Dashboard reflects: Agendado → Publicado (manually confirmed by manager) 7. MAIN DASHBOARD Screen: "Dashboard" KPI cards: - Total active pieces - Pending client approval - In revision (Em Ajuste) - Scheduled - Published Campaign list with: - Campaign name - Client - Progress bar (pieces by status) - Quick action button Quick-access button: "Nova Campanha" (New Campaign) --- STATUS SYSTEM (use colored badges in pt-BR) - "Em Briefing" — gray - "Em Produção" — blue - "Revisão Interna" — amber - "Aguardando Cliente" — amber - "Em Ajuste" — red - "Aprovado" — green - "Agendado" — blue - "Publicado" — green - "Reprovado" — red --- UI/UX DESIGN DIRECTION - Design language: clean, modern, B2B SaaS — similar to Notion, Linear, or Figma - Color palette: neutral background (off-white #FAF9F5), dark ink (#1A1A18), minimal color accents only for status badges - Typography: DM Sans or Lato — clean, readable, professional - Layout: sidebar navigation (left) + main content area (right) - Components: cards, status badges, progress bars, drag-and-drop upload areas, modal dialogs - Mobile-responsive is a bonus, not required for MVP --- NAVIGATION STRUCTURE (sidebar) - Dashboard (Painel) - Campanhas (Campaigns) - Peças (Content Pieces) - Clientes (Clients) - Configurações (Settings) --- DATA MODELS (simplified) Campaign: - id, name, client_id, start_date, end_date, objective, networks[], status, created_by, created_at ContentPiece: - id, campaign_id, network, format, publication_date, assigned_social_media, assigned_designer, copy_caption, copy_cta, copy_hashtags, artwork_url, status, approval_timestamp, approval_user, approval_ip, created_at ContentVersion: - id, piece_id, version_number, artwork_url, copy_caption, copy_cta, copy_hashtags, status, client_comment, submitted_at, submitted_by Client: - id, name, logo_url, contact_email, approval_link_token User: - id, name, email, role (planner | social_media | designer | manager | admin), account_id --- SECURITY AND LGPD COMPLIANCE - All routes must be authenticated (except client approval portal) - Role-based access control (RBAC): users only see and edit what their role permits - Approval records are immutable — no update or delete allowed after confirmation - All file uploads: validate file type and size before storing - Use HTTPS everywhere - Log all approval actions with timestamp, user ID, and IP address --- FINANCIAL CONSIDERATIONS (architecture decisions) - Implement file size limits per upload (images: max 10MB, videos: max 100MB) - Store files in cloud object storage (not local filesystem) - Track storage usage per account for future billing enforcement - Client approval portal accessible via unique shareable link 5. File upload and cloud storage integration 6. Status workflow system with transitions 7. Version history per content piece 8. Responsive sidebar navigation Include demo seed data: - 1 demo agency account - 3 demo users (manager, social media, designer) - 1 demo client ("Empresa Demo") - 2 demo campaigns with pieces in different statuses - Example mockups for Instagram and LinkedIn --- START BY: 1. Setting up the project structure 2. Creating the database schema 3. Building the authentication system 4. Building the Dashboard screen 5. Building the Campaign creation screen 6.
