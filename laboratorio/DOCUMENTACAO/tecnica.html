Arquitetura

O BBrain é uma aplicação monolítica simples. Não há frameworks, ORMs ou dependências externas além da googleapis.

Browser (cliente)
    │
    ↓ HTTP (JSON REST)
    │
Fly.io — server.js  ← Node.js nativo, porta 3000
    ├── → Google Sheets API v4  ← banco de dados
    └── → Gmail SMTP (465/TLS)  ← emails de reset

Autenticação

Mecanismo

A senha é armazenada como SHA-256 hash na variável de ambiente BBRAIN_PASSWORD_HASH. Nunca há texto puro em disco ou memória.

Token de sessão

Após login bem-sucedido, o servidor gera um token diário (SHA-256 do hash + data atual). O token é salvo no localStorage do browser e enviado no header Authorization: Bearer <token> em todas as requisições.

Expiração

O token expira automaticamente à meia-noite (baseado em data UTC). O usuário é redirecionado para a tela de login sem interrupção.

Service Account

Acesso à API via service account: aproove-server@aproove.iam.gserviceaccount.com. As credenciais JSON ficam na variável de ambiente GOOGLE_CREDENTIALS_JSON.

Deploy

Requisitos

Fly.io CLI instalado (flyctl), autenticado com a conta Aproove. O arquivo fly.toml deve estar na raiz do projeto.

Comando

fly deploy

Fluxo

O Fly.io detecta automaticamente Node.js via Dockerfile ou buildpack. A aplicação é implantada na região GRU (São Paulo) no free tier, com uma máquina compartilhada de 256MB RAM.
