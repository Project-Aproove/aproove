You are a senior full-stack developer and UI/UX designer tasked with building the MVP of "Aproove" — a B2B SaaS platform for social media content approval workflow management.

IMPORTANT: All user interface text, labels, buttons, messages, and content must be written in Brazilian Portuguese (pt-BR).

---

PROJECT OVERVIEW
Aproove centralizes the content approval flow between marketing teams and their clients. The core problem it solves: today, content approvals happen via WhatsApp, email, and PDFs — causing rework, delays, misalignment, and publication errors.

---

TECH STACK
- Frontend: React + TypeScript + Tailwind CSS
- Backend: Node.js + Express (or Next.js API routes)
- Database: Supabase (PostgreSQL) or Firebase
- Storage: Supabase Storage or Cloudinary (for image/video uploads)
- Auth: Supabase Auth or Clerk
- Deployment: Replit (web)

---

USER ROLES
1. Internal Team (authenticated):
   - Planner: creates campaigns, defines networks/formats/dates
   - Social Media: writes copy (caption, CTA, hashtags) per piece
   - Designer: uploads artwork per piece
   - Marketing Manager: internal review and approval
   - Admin: manages users, clients, integrations

2. External User (client):
   - Client Approver: accesses via public link or simple login, views mockup, approves or requests changes

---

CORE FEATURES TO BUILD (MVP SCOPE)

1. CAMPAIGN PLANNING
   Screen: "Nova Campanha" (New Campaign)
   Fields:
   - Campaign name
   - Associated client
   - Start and end date
   - Campaign objective (dropdown: Awareness / Engagement / Lead Generation / Institutional / Commercial)
   - Social networks (multi-select checkboxes with icons): Instagram, LinkedIn, TikTok, Facebook, X/Twitter
   - For each selected network: format(s) + publication date + assigned social media user + assigned designer
   
   On save: system auto-generates a list of content pieces with status "Em Briefing" (In Briefing)

2. CONTENT PRODUCTION
   Screen: "Detalhe da Peça" (Content Piece Detail)
   Fields:
   - Social network + format (displayed, not editable)
   - Publication date
   - Status badge
   - Copy area: caption, CTA, hashtags (filled by Social Media role)
   - Artwork upload (drag & drop, image or video) — filled by Designer role
   - Artwork preview after upload
   - Button: "Marcar como pronto para revisão" (Mark as ready for review)

3. VISUAL MOCKUP GENERATION
   When both copy and artwork are complete, the system renders a visual mockup simulating the post as it would appear on the social network:
   - Instagram: white phone frame, profile picture placeholder, username placeholder, image, caption, like/comment bar
   - LinkedIn: feed-style card with profile photo, name, text, image
   - TikTok: vertical dark phone frame with overlay text, like/share buttons
   - Facebook: standard feed post card
   - X/Twitter: tweet card with text and image

4. CLIENT APPROVAL PORTAL
   Screen: public or lightly-authenticated page accessible via unique link
   Layout:
   - Agency logo (customizable) at top
   - Campaign name + scheduled publication date
   - Visual mockup (network-specific frame)
   - Caption text displayed below
   - Three action buttons: [✓ Aprovar] [✎ Solicitar Ajuste] [✗ Reprovar]
   - Comment field (mandatory, appears when "Solicitar Ajuste" is clicked)
   - After approval: buttons disappear, confirmation message shown
   
   On approval: timestamp, user identifier, and IP are recorded (immutable)
   On revision request: comment is attached to the piece; status changes to "Em Ajuste"

5. VERSION HISTORY
   Screen: "Histórico de Versões" (Version History)
   - Vertical timeline per content piece
   - Each version shows: version number, date, who submitted, artwork thumbnail, copy text, client's comment (if any), status badge
   - Current version highlighted

6. AUTOMATIC SCHEDULING
   Logic (backend):
   - On client approval, system reads the scheduled publication date from campaign planning
   - Status updates automatically: Aprovado → Agendado
   - (For MVP: scheduling confirmation is visual only — no real API integration with social networks yet)
   - Dashboard reflects: Agendado → Publicado (manually confirmed by manager)

7. MAIN DASHBOARD
   Screen: "Dashboard"
   KPI cards:
   - Total active pieces
   - Pending client approval
   - In revision (Em Ajuste)
   - Scheduled
   - Published
   
   Campaign list with:
   - Campaign name
   - Client
   - Progress bar (pieces by status)
   - Quick action button
   
   Quick-access button: "Nova Campanha" (New Campaign)

---

SECURITY AND LGPD COMPLIANCE
- All routes must be authenticated (except client approval portal)
- Role-based access control (RBAC): users only see and edit what their role permits
- Approval records are immutable — no update or delete allowed after confirmation
- All file uploads: validate file type and size before storing
- Use HTTPS everywhere
- Log all approval actions with timestamp, user ID, and IP address

---

FINANCIAL CONSIDERATIONS (architecture decisions)
- Implement file size limits per upload (images: max 10MB, videos: max 100MB)
- Store files in cloud object storage (not local filesystem)
- Track storage usage per account for future billing enforcement
- Implement soft limits with warning notifications (80% of plan limit)
- Hard limits that block uploads at 100% of plan storage
